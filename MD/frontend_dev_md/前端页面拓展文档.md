## 二、AI 问答助手页 `/ai/chat`

**功能来源**：README 5.3 AI问答助手模块

**依赖后端接口**

| 接口 | 说明 |
|------|------|
| `POST /api/ai/chat` | 发送消息，获取 AI 回复 |
| `GET /api/ai/chat/history` | 获取历史对话记录 |
| `POST /api/ai/chat/clear` | 清空当前会话历史 |

**布局草图（参考常见 AI 对话界面）**

```
┌─────────────────────────────────────────────────────┐
│  AI 编程助手                          [ 清空对话 ]    │
├─────────────────────────────────────────────────────┤
│                                                     │
│  ┌──────────────────────────────────────────────┐  │
│  │                                              │  │
│  │  [用户头像]  两数之和怎么用哈希表解？         │  │
│  │                                              │  │
│  │  [AI头像]  哈希表解法的核心思路是：...        │  │
│  │           步骤一：...                        │  │
│  │           步骤二：...                        │  │
│  │                                              │  │
│  │  [用户头像]  能给我看个 Java 示例吗？         │  │
│  │                                              │  │
│  │  [AI头像]  当然，以下是 Java 示例代码：...    │  │
│  │                                              │  │
│  └──────────────────────────────────────────────┘  │
│                                                     │
│  ┌──────────────────────────────────┐  [ 发 送 ]   │
│  │  输入你的问题...                  │              │
│  └──────────────────────────────────┘              │
└─────────────────────────────────────────────────────┘
```

**交互要点**

- AI 回复支持 Markdown 渲染（代码块高亮）
- 发送后显示加载动画；若后端支持 SSE 流式输出，需在首帧检测业务错误码（见前端方案设计.md §八 SSE 处理规范）
- **限流触发时（后端返回 `code: 42900`）**：不弹通用 Toast，取响应体 `message` 字段内容展示给用户（如「您今日的 AI 问答次数已用完」）；触发后输入框禁用，用户刷新页面或次日后自动恢复——**不再主动拉取 `ai.rate_limit.user_daily` 配置**（该字段已从 ai_config 中移除）
- 历史记录按 `chat_id` 分组，支持新建会话
- AI 功能全局开关关闭时，显示「AI 功能暂时不可用」提示并禁用输入框

## 三、AI 代码分析页 `/ai/code-analysis`

**功能来源**：README 5.2 AI代码智能分析与判题模块

**入口方式**：做题页提交成功后，结果区域出现「AI 分析」按钮 → 跳转本页

**依赖后端接口**

| 接口 | 说明 |
|------|------|
| `POST /api/ai/code/analysis` | 提交代码进行 AI 分析 |
| `GET /api/ai/code/history` | 获取用户历史分析记录 |

**布局草图**

```
┌──────────────────────────┬─────────────────────────────┐
│  左侧：题目信息 + 代码     │  右侧：AI 分析结果            │
│                          │                             │
│  题目：两数之和            │  ┌─────────────────────────┐│
│  语言：Java               │  │  代码风格评分：8/10        ││
│                          │  │  代码质量评分：7/10        ││
│  ┌──────────────────────┐│  │                         ││
│  │  用户代码（只读）       ││  │  错误分析：               ││
│  │                      ││  │  1. 第 12 行存在...       ││
│  │                      ││  │  2. 建议改为...           ││
│  │                      ││  │                         ││
│  └──────────────────────┘│  │  优化建议：               ││
│                          │  │  结合哈希表考点...         ││
│  [ 重新分析 ]             │  └─────────────────────────┘│
└──────────────────────────┴─────────────────────────────┘
```

**交互要点**

- 左侧代码为只读 Monaco Editor（方便用户对照）
- 右侧分析结果为 Markdown 渲染
- AI 分析为异步操作，显示加载动画（预计 3-10s）
- 分析结果可「复制」「分享」

## 七、AI 配置管理页 `/admin/ai-config`（仅管理员）

**功能来源**：README 5.6 系统配置与AI开关模块

**依赖后端接口**

| 接口 | 说明 |
|------|------|
| `GET /api/admin/ai/config` | 获取当前 AI 配置列表（仅含 AI 功能配置，不含限流规则） |
| `POST /api/admin/ai/config` | 修改 AI 配置 |

> **注**：限流规则（调用次数上限等）由独立的限流管理页维护，本页不再承载限流配置，可通过页面底部的「→ 前往限流规则管理」入口跳转至 `/admin/rate-limit`。

**布局草图**

```
┌──────────────────────────────────────────────────────────┐
│  AI 系统配置                                              │
├──────────────────────────────────────────────────────────┤
│  AI 功能全局开关     [ ● 开启 / ○ 关闭 ]                  │
├──────────────────────────────────────────────────────────┤
│  大模型配置（阿里百炼 Model Studio）                       │
│  API 密钥          [ ******* ]  [ 修改 ]                 │
│  API 端点          [ https://dashscope.aliyuncs.com/... ] │
│  聊天模型名称       [ qwen-plus ]（可选：qwen-max）        │
│  嵌入模型名称       [ text-embedding-v3 ]                 │
│  ⚠️ 修改嵌入模型后需重新向量化全量数据，请谨慎操作           │
├──────────────────────────────────────────────────────────┤
│  RAG 检索配置                                             │
│  检索条数（topK）   [ 3      ]                            │
│  相似度阈值         [ 0.7    ]                            │
├──────────────────────────────────────────────────────────┤
│                                          [ 保 存配置 ]    │
│  ─────────────────────────────────────────────────────  │
│  → 前往限流规则管理                                       │
└──────────────────────────────────────────────────────────┘
```

**配置字段与 ai_config 键映射**

| 界面字段 | config_key | 说明 |
|---------|-----------|------|
| AI 全局开关 | `ai.global.enable` | `true`/`false`，关闭后前端隐藏所有 AI 入口 |
| API 密钥 | `ai.model.api_key` | 百炼平台 API Key，展示时脱敏（`****`） |
| API 端点 | `ai.model.base_url` | 百炼 OpenAI 兼容端点，通常无需修改 |
| 聊天模型名称 | `ai.model.name` | 可选值：`qwen-turbo` / `qwen-plus` / `qwen-max` |
| 嵌入模型名称 | `ai.model.embedding_name` | 默认 `text-embedding-v3`，修改需重建向量索引 |
| 检索条数 | `ai.rag.top_k` | 整数，建议 3-5 |

## 八、限流规则管理页 `/admin/rate-limit`（仅管理员）

**功能来源**：`rate_limit_dev.md` 第六节 管理员动态配置接口

**依赖后端接口**

| 接口 | 说明 |
|------|------|
| `GET /api/admin/rate-limit/rules` | 获取所有限流规则列表 |
| `POST /api/admin/rate-limit/rule/update` | 更新指定规则（修改后实时生效，无需重启） |

**布局草图**

```
┌──────────────────────────────────────────────────────────────┐
│  限流规则管理                       [ 刷新缓存（预热） ]        │
├─────────────────┬──────┬────────┬──────────┬────────────────┤
│  规则名称        │ 窗口  │ 最大次数 │ 状态      │ 操作           │
├─────────────────┼──────┼────────┼──────────┼────────────────┤
│ 用户提交-分钟级  │ 60s  │   5    │ ● 启用   │ [编辑]         │
│ 用户提交-每日    │ 24h  │  200   │ ● 启用   │ [编辑]         │
│ 用户同题冷却     │ 30s  │   1    │ ● 启用   │ [编辑]         │
│ IP提交-分钟级    │ 60s  │   10   │ ● 启用   │ [编辑]         │
│ 全局提交-秒级    │  1s  │   50   │ ● 启用   │ [编辑]         │
│ AI问答-每日（扩展）│ 24h │   50   │ ○ 禁用   │ [编辑]         │
└─────────────────┴──────┴────────┴──────────┴────────────────┘
```

**编辑弹窗（点击「编辑」触发）**

```
┌────────────────────────────────────────┐
│  编辑限流规则 - 用户提交-分钟级           │
├────────────────────────────────────────┤
│  时间窗口（秒）  [ 60  ]               │
│  最大次数        [ 5   ]               │
│  是否启用        [ ● 启用 / ○ 禁用 ]   │
├────────────────────────────────────────┤
│  ⚠️ 修改后立即生效，无需重启服务          │
│              [ 取消 ]  [ 确认修改 ]    │
└────────────────────────────────────────┘
```

**交互要点**

- 「刷新缓存（预热）」按钮操作需弹确认框（「确认重新预热所有规则缓存？」），成功后 Toast 提示「缓存预热完成」
- 编辑保存后：行内状态立即刷新（数值/状态更新），Toast 提示「规则已更新，立即生效」
- `is_enable` 为禁用时，对应规则行灰显（`opacity: 0.5`），状态标签显示「○ 禁用」
- 规则名称只读，不允许修改（`rule_key` 是唯一键，前端不提供修改入口）
- 操作失败时 Toast 显示后端返回的 `message`
