## 一、整体设计原则

| 原则 | 说明 |
|------|------|
| 简洁克制 | 无多余装饰，内容优先，以白色/浅灰为主色调 |
| 功能集中 | 每个页面聚焦单一任务，减少跳转层级 |
| 状态清晰 | 判题状态、登录状态、加载状态均有明确的视觉反馈 |
| 权限分层 | 普通用户/管理员两套菜单，后台页面独立路由 |
| 响应式布局 | 优先适配 1280px 以上桌面端，兼容 768px 以上平板端 |

### 5.1 题目列表页 `/questions`

**参考 LeetCode 题库页面**

#### 布局

```
┌──────────────────────────────────────────────────────────┐
│  搜索框（按标题模糊搜索）    标签筛选（多选下拉）  [搜索]  │
├────┬──────────────────┬──────────┬──────────┬────────────┤

├────┼──────────────────┼──────────┼──────────┼────────────┤
│ 1  │ 两数之和          │ 哈希表    │  68.2%   │ 12,453     │
│ 2  │ ...              │ ...      │  ...     │  ...       │
```

#### 字段来源

- 数据来源：`POST /api/question/list/page/vo`，返回 `Page<QuestionVO>`

#### 交互细节

- 搜索框输入后点击「搜索」或回车触发查询，不实时搜索（减少请求）
- 标签从已有题目中枚举（前端维护常用标签列表，或后续接口支持）
- 每页 20 条，分页组件
- 点击题目标题跳转 `/view/question/:id`
- 未登录也可浏览列表（接口要求需登录，需先引导登录）

#### 布局（左右分屏）

```
┌──────────────────────────┬────────────────────────────────┐
│  左侧：题目信息面板        │  右侧：代码编辑区               │
│  ┌────────────────────┐  │  ┌────────────────────────────┐│
│  │ 标题  #ID           │  │  │ 语言选择：[Java ▾]          ││

│  ├────────────────────┤  │  │  （代码编辑器）               ││
│  │ 题目描述（Markdown） │  │  │                            ││
│  │                    │  │  │                            ││
│  │                    │  │  └────────────────────────────┘│
│  │                    │  │  ┌────────────────────────────┐│
│  │                    │  │  │ 判题配置：时间限制 / 内存限制  ││
│  │                    │  │  └────────────────────────────┘│
│  │                    │  │  [ 提交代码 ]                   │
│  └────────────────────┘  │                                │
│  ┌────────────────────┐  │  ─────── 提交结果区 ───────────  │
│  │ 我的提交记录（最近5条）│  │  ┌────────────────────────────┐│
│  │ 通过 Java 2ms 8KB  │  │  │ 判题状态：✅ 通过             ││
│  │ 错误 Java  -   -   │  │  │ 执行用时：2ms                ││
│  └────────────────────┘  │  │ 内存消耗：8.2MB              ││
│                          │  └────────────────────────────┘│
└──────────────────────────┴────────────────────────────────┘
```

#### 字段来源

**题目信息**

| 展示内容 | 来源字段 | 接口 |
|---------|---------|------|
| 题目标题 | `QuestionVO.title` | `GET /api/question/get/vo?id=xxx` |
| 题目描述 | `QuestionVO.content`（Markdown 渲染） | 同上 |
| 标签 | `QuestionVO.tags`（已反序列化为 List） | 同上 |
| 提交数 | `QuestionVO.submitNum` | 同上 |
| 通过数 | `QuestionVO.acceptedNum` | 同上 |
| 时间限制 | `QuestionVO.judgeConfig.timeLimit`（ms） | 同上 |
| 内存限制 | `QuestionVO.judgeConfig.memoryLimit`（KB） | 同上 |

**代码提交**

| 字段 | 说明 |
|------|------|
| `language` | 语言选择下拉，枚举：`java` / `c++` / `golang` |
| `code` | 编辑器内容 |
| `questionId` | 当前题目 ID |
| 接口 | `POST /api/question_submit/`，请求体为 `QuestionSubmitAddRequest` |

**提交响应处理（三条分支）**

```
点击「提交代码」
    │
    ├─ 返回 code: 42900（限流触发）
    │     → 不进入轮询，直接在结果区展示限流提示
    │     → 提示文案取自响应体 message 字段（后端已区分粒度）
    │     → 根据 message 内容区分展示：
    │         · 含「今日」→ 「您今日提交次数已达上限，明天再来吧」
    │         · 含「同一题目」→ 「请等待 xx 秒后再提交该题」
    │         · 全局繁忙 → 「系统当前繁忙，请稍后再试」
    │     → 提交按钮进入禁用态，同题冷却场景可展示倒计时
    │     → 前端本地记录限流触发时间，冷却期内拦截重复请求（不再发起网络请求）
    │
    ├─ 返回 code: 0（提交成功，获得提交记录 ID）
    │     → 进入轮询流程（见下方）
    │
    └─ 其他错误（网络/500）
          → Toast 通用错误提示
```

**提交后轮询判题状态**

- 提交成功后返回提交记录 ID（`Long`）
- 每隔 1.5s 查询 `POST /api/question_submit/list/page`，过滤 `questionId` + 当前用户最新一条记录
- 状态为 `0（等待中）`或 `1（判题中）` 时继续轮询，变为 `2（成功）`或 `3（失败）` 时展示结果
- 展示 `judgeInfo.message`（判题结果消息）、`judgeInfo.time`（ms）、`judgeInfo.memory`（KB）
- **注意**：限流错误（42900）不属于判题失败，不在判题结果区展示「失败」标签，而是在提交按钮区域展示独立的限流提示

**判题结果消息映射**

| message 值 | 展示文字 | 颜色 |
|-----------|---------|------|
| Accepted | 通过 | 绿色 |
| Wrong Answer | 答案错误 | 红色 |
| Time Limit Exceeded | 超出时间限制 | 黄色 |
| Memory Limit Exceeded | 超出内存限制 | 黄色 |
| Compile Error | 编译错误 | 黄色 |
| Runtime Error | 运行时错误 | 红色 |
| System Error | 系统错误 | 红色 |

**我的提交记录（左侧下方）**

- 来源：`POST /api/question_submit/list/page`，传入 `questionId` 当前题目
- 展示最近 5 条：状态、语言、时间、用时（`judgeInfo.time`）、内存（`judgeInfo.memory`）
- 点击条目可展开查看代码（仅自己的记录可查看代码）

#### 代码编辑器

- 推荐使用 `@monaco-editor/vue3`（Monaco Editor，与 VSCode 同款）
- 初始代码模板（各语言默认模板，前端硬编码）：
  - Java：`class Solution { public static void main(String[] args) { } }`
  - C++：`#include <bits/stdc++.h>\nusing namespace std;\nint main() { return 0; }`
  - Golang：`package main\nimport "fmt"\nfunc main() { }`
- 语言切换时提示是否清空代码

## 八、接口请求封装说明

- 所有接口基于已生成的 `generated/services/*.ts`，通过 `OpenAPI.BASE = 'http://localhost:8101'` 配置
- 统一在 `src/plugins/axios.ts` 处理全局错误，请求携带 Cookie（`WITH_CREDENTIALS: true`，已配置）

### 全局响应拦截策略

> **重要**：后端所有业务错误均以 **HTTP 200 + 响应体中的 `code` 字段** 返回，不依赖 HTTP 状态码（4xx/5xx），拦截器需在响应体层面判断，不能仅依赖 `axios` 的 `error interceptor`。

| 错误类型 | 判断方式 | 处理策略 |
|---------|---------|---------|
| 未登录 | `code === 40100` | 跳转 `/user/login`，清空本地用户状态 |
| 无权限 | `code === 40101` | 跳转 `/noAuth` |
| 禁止访问 | `code === 40300` | Toast 提示「无访问权限」 |
| 资源不存在 | `code === 40400` | Toast 提示「内容不存在或已删除」 |
| **限流触发** | **`code === 42900`** | **不弹通用错误，将 `message` 原文透传给调用方**，由各页面自行展示（如提交页展示倒计时、AI 问答页禁用输入框） |
| 参数错误 | `code === 40000` | Toast 提示响应体 `message`（通常是参数校验失败原因） |
| 服务器异常 | `code === 50000` / HTTP 5xx | Toast 提示「服务器开小差了，请稍后再试」 |

```typescript
// src/plugins/axios.ts（响应拦截器示意）
axios.interceptors.response.use(
  (response) => {
    const { code, message } = response.data;
    if (code === 0) return response;           // 业务成功，放行
    if (code === 40100) { router.push('/user/login'); return Promise.reject(response.data); }
    if (code === 40101) { router.push('/noAuth'); return Promise.reject(response.data); }
    if (code === 42900) {
      // 限流错误：不弹通用 Toast，直接 reject，由调用方处理 message
      return Promise.reject({ isRateLimit: true, message });
    }
    // 其他业务错误：弹 Toast 显示 message
    Message.error(message || '操作失败');
    return Promise.reject(response.data);
  },
  (error) => {
    // HTTP 层错误（网络超时、5xx 等）
    Message.error('网络异常，请检查网络连接');
    return Promise.reject(error);
  }
);
```

### SSE 流式输出的错误处理（AI 问答场景）

若 AI 问答接口采用 SSE 流式输出，标准 axios 拦截器无法覆盖，需在 SSE 客户端层单独处理：

- **连接建立失败（HTTP 非 200）**：在 `EventSource` 的 `onerror` 中捕获，提示「AI 服务暂时不可用」
- **首帧含业务错误**：约定 SSE 流的第一个 `data` 事件若含 `{"code": 42900, ...}`，前端需立即关闭连接并展示限流提示
- **流式输出完成**：约定服务端推送 `data: [DONE]` 事件作为结束信号，前端收到后关闭连接
- **用户主动取消**：点击「停止生成」时调用 `eventSource.close()`，UI 恢复可输入状态
