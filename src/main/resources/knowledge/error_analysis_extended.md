tag: 贪心
title: 贪心-WA-局部最优不等于全局最优

【常见原因】
贪心策略选择错误，局部最优解无法推导出全局最优解。典型反例：硬币找零问题在非标准面额（如 1, 3, 4）下，贪心选最大面额会得到错误结果（如凑 6 = 4+1+1 而非 3+3）。

【排查步骤】
- 尝试构造反例验证贪心策略的正确性
- 如果无法严格证明贪心正确，改用动态规划
- 常见的正确贪心场景：区间调度按结束时间排序、Huffman 编码、Dijkstra 最短路径

content_type: 错因分析
tag: 回溯
title: 回溯-TLE-未有效剪枝导致搜索空间爆炸

【常见原因】
回溯搜索的时间复杂度是指数级的，如果不加剪枝，n 稍大就会超时。常见遗漏：未排序导致无法跳过重复分支、未提前判断剩余元素不足、未利用约束条件缩小搜索范围。

【排查步骤】
- 检查是否对输入排序以便跳过重复元素
- 检查是否在递归入口处判断"剩余元素是否足够凑出目标"
- 对于约束满足问题（数独、N皇后），检查是否用位运算或数组快速判断冲突

content_type: 错因分析
tag: 滑动窗口
title: 滑动窗口-WA-窗口收缩条件或更新时机错误

【常见原因】
1. 收缩条件写反：应该在窗口不满足条件时收缩，但写成了满足条件时收缩
2. 更新答案的时机错误：求最长时应在窗口合法时更新，求最短时应在窗口刚好满足条件时更新
3. 字符频率计数未正确维护：收缩窗口时忘记减少左端字符的计数

【排查步骤】
- 明确窗口的合法条件是什么
- 区分"最长合法窗口"和"最短满足条件窗口"的模板差异
- 用小样例手动模拟左右指针的移动过程

content_type: 错因分析
tag: 前缀和
title: 前缀和-WA-下标偏移导致区间计算错误

【常见原因】
前缀和数组 preSum 的长度为 n+1，preSum[0] = 0，preSum[i] = arr[0] + ... + arr[i-1]。区间 [l, r] 的和为 preSum[r+1] - preSum[l]。常见错误是下标偏移搞混，导致多算或少算一个元素。

【排查步骤】
- 确认 preSum 数组的定义：preSum[i] 表示前 i 个元素的和还是前 i-1 个
- 用长度为 3 的小数组手动验证区间和公式
- 二维前缀和的容斥公式更容易出错，务必画图确认

content_type: 错因分析
tag: 链表
title: 链表-RE-空指针访问导致NullPointerException

【常见原因】
1. 遍历时未检查 node.next 是否为 null 就访问 node.next.next
2. 删除节点后继续使用已删除节点的引用
3. 反转链表时指针赋值顺序错误导致链表断裂，后续访问到 null

【排查步骤】
- 在访问 node.next 或 node.val 之前始终检查 node != null
- 使用虚拟头节点（dummy）统一处理头节点的特殊情况
- 反转链表时画图标注每一步的指针变化

content_type: 错因分析
tag: 二叉树
title: 二叉树-WA-递归返回值处理不当

【常见原因】
1. 递归函数的返回值语义不清晰：有时返回"以当前节点为根的子树的结果"，有时返回"经过当前节点的路径结果"，混淆导致错误
2. 忘记处理空节点的 base case
3. 路径问题中混淆"根到叶路径"和"任意节点到任意节点路径"

【排查步骤】
- 明确递归函数的返回值含义，写在注释中
- 确认 base case 覆盖了 null 节点和叶子节点
- 对于路径问题，区分"单边路径"（向下）和"折线路径"（经过某节点向两边延伸）

content_type: 错因分析
tag: 堆
title: 堆-WA-大小堆选择错误或比较器写反

【常见原因】
1. 求前 K 大用最小堆（堆顶是第 K 大），求前 K 小用最大堆（堆顶是第 K 小），容易搞反
2. 自定义比较器方向写反：(a, b) -> a - b 是最小堆，(a, b) -> b - a 是最大堆
3. 比较器中 a - b 在极端值时可能整数溢出，应改用 Integer.compare(a, b)

【排查步骤】
- 明确需要最小堆还是最大堆，写出堆顶元素的含义
- 用 Integer.compare 替代减法避免溢出
- 检查堆的大小维护逻辑：何时入堆、何时弹出

content_type: 错因分析
tag: 字符串
title: 字符串-TLE-频繁拼接导致O(n²)时间复杂度

【常见原因】
Java 中 String 是不可变对象，每次 += 拼接都会创建新的 String 对象并复制原有内容。在循环中拼接 n 次，总时间为 O(1+2+...+n) = O(n²)。当 n ≥ 10^4 时明显超时。

【正确做法】
使用 StringBuilder 进行字符串拼接，最后调用 toString()。StringBuilder.append() 是 O(1) 均摊时间，总时间 O(n)。Python 中类似问题用 ''.join(list) 替代循环拼接。

content_type: 错因分析
tag: 取模
title: 取模运算-WA-中间结果溢出或负数取模

【常见原因】
1. 乘法后再取模，但乘法本身已经溢出 int/long 范围。应该在每次乘法后立即取模：(a * b) % MOD 改为 ((a % MOD) * (b % MOD)) % MOD
2. 减法取模可能产生负数：(a - b) % MOD 可能为负，应改为 ((a - b) % MOD + MOD) % MOD
3. 除法不能直接取模，需要用模逆元：a / b % MOD = a * b^(MOD-2) % MOD（费马小定理，要求 MOD 为质数）

【排查步骤】
- 检查所有中间计算是否在乘法后立即取模
- 检查减法取模是否处理了负数情况
- 除法取模必须转换为乘以逆元
