tag: 滑动窗口
title: 子数组/子串问题的题型识别与算法选择

【题型特征】
题目要求在连续子数组或子串中找满足某条件的最长/最短/计数结果。关键词包括：连续子数组、子串、窗口、不重复、最长、最短、恰好包含。

【推荐算法】
1. 滑动窗口（双指针）：维护左右指针 [left, right]，right 扩展窗口直到不满足条件，left 收缩窗口恢复条件。适用于"最长满足条件的子数组"类问题，时间复杂度 O(n)。
2. 前缀和：当问题涉及子数组求和时，preSum[j] - preSum[i] 可以 O(1) 计算任意区间和。配合哈希表可解"和为 K 的子数组个数"。
3. 单调队列：滑动窗口内求最大/最小值时，用双端队列维护单调性，O(n) 解决。

【判断依据】
- 如果条件具有单调性（窗口越大越容易/越难满足）→ 滑动窗口
- 如果需要精确区间和 → 前缀和
- 如果窗口固定大小且需要极值 → 单调队列

content_type: 题型模式
tag: 动态规划
title: 最优子结构问题的题型识别与算法选择

【题型特征】
题目要求最大/最小/方案数，且问题可以分解为子问题，子问题的最优解能组合成原问题的最优解。关键词包括：最少步数、最大利润、方案总数、能否到达、最长递增。

【推荐算法】
动态规划（DP）：定义状态 dp[i] 表示到第 i 个元素为止的最优解，通过状态转移方程从已知子问题推导。常见模式：
1. 线性 DP：dp[i] 只依赖 dp[i-1] 或 dp[i-2]，如爬楼梯、打家劫舍
2. 背包 DP：选/不选模型，dp[i][j] 表示前 i 个物品容量为 j 的最优解
3. 区间 DP：dp[i][j] 表示区间 [i,j] 的最优解，如矩阵链乘法、戳气球
4. 树形 DP：在树结构上自底向上递推

【判断依据】
- 问题有重叠子问题 + 最优子结构 → DP
- 如果只需判断可行性（能否到达）→ DP 或 BFS
- 如果需要具体方案路径 → DP + 回溯记录路径

content_type: 题型模式
tag: 回溯
title: 排列/组合/子集问题的题型识别与算法选择

【题型特征】
题目要求枚举所有可能的排列、组合、子集或满足条件的方案。关键词包括：所有排列、所有组合、所有子集、全排列、N皇后、数独、路径搜索。

【推荐算法】
回溯法：通过递归逐步构建候选解，在发现当前路径不可能产生有效解时立即回退（剪枝）。模板：
1. 选择：将当前元素加入路径
2. 递归：进入下一层决策
3. 撤销：回退当前选择，尝试其他分支

【剪枝技巧】
- 排序后跳过重复元素（去重组合）
- 记录已使用元素（排列问题用 visited 数组）
- 提前判断剩余元素是否足够（组合问题的长度剪枝）
- 约束传播（数独、N皇后中提前排除不可能的位置）

content_type: 题型模式
tag: BFS,DFS
title: 图/网格搜索问题的题型识别与算法选择

【题型特征】
题目涉及图、网格、迷宫、岛屿等结构，要求最短路径、连通分量、可达性判断。关键词包括：最短路径、最少步数、岛屿数量、连通分量、层序遍历。

【推荐算法】
1. BFS（广度优先搜索）：适用于无权图最短路径、层序遍历。用队列实现，每次扩展一层，第一次到达目标即为最短路径。
2. DFS（深度优先搜索）：适用于连通性判断、路径存在性、岛屿计数。用递归或栈实现，遍历所有可达节点。
3. Dijkstra：带权图最短路径，用优先队列实现，适用于非负权边。

【判断依据】
- 无权图求最短距离 → BFS
- 判断连通性/计数连通分量 → DFS 或 BFS 均可
- 带权图最短路径 → Dijkstra
- 需要遍历所有路径 → DFS + 回溯

content_type: 题型模式
tag: 贪心
title: 区间/调度问题的题型识别与算法选择

【题型特征】
题目涉及区间合并、活动选择、任务调度、跳跃游戏等，要求最优策略。关键词包括：最少区间覆盖、最多不重叠区间、能否跳到终点、最少硬币数。

【推荐算法】
贪心：每一步选择当前看起来最优的方案，且局部最优能推导出全局最优。常见模式：
1. 区间调度：按结束时间排序，优先选结束早的（最多不重叠区间）
2. 区间合并：按起始时间排序，依次合并重叠区间
3. 跳跃贪心：维护当前能到达的最远位置

【注意事项】
- 贪心必须能证明正确性（局部最优 → 全局最优），否则应考虑 DP
- 常见反例：硬币找零问题在非标准面额下贪心不成立，需要 DP

content_type: 题型模式
tag: 二分查找
title: 查找/有序数组问题的题型识别与算法选择

【题型特征】
题目涉及有序数组查找、满足条件的最小/最大值、分配问题的最优解。关键词包括：有序数组、查找目标、最小化最大值、最大化最小值、分割数组。

【推荐算法】
1. 标准二分查找：在有序数组中查找目标值或插入位置，O(log n)
2. 二分答案：当答案具有单调性时（答案越大越容易/越难满足条件），对答案范围二分，每次用 O(n) 验证可行性

【判断依据】
- 有序数组精确查找 → 标准二分
- "最小化最大值"或"最大化最小值" → 二分答案
- 答案范围明确且可验证 → 二分答案

content_type: 题型模式
tag: 堆,优先队列
title: Top-K/合并问题的题型识别与算法选择

【题型特征】
题目要求前 K 大/小元素、第 K 大元素、合并 K 个有序链表/数组、数据流中位数。关键词包括：第K大、前K个、合并K个有序、数据流、中位数。

【推荐算法】
1. 最小堆/最大堆：维护大小为 K 的堆，O(n log K) 找前 K 大/小
2. 快速选择（QuickSelect）：基于快排 partition，平均 O(n) 找第 K 大
3. 多路归并：用最小堆合并 K 个有序序列，每次取堆顶最小元素

【判断依据】
- 前 K 大 → 最小堆（堆顶是第 K 大）
- 前 K 小 → 最大堆（堆顶是第 K 小）
- 只需第 K 大不需排序 → 快速选择更优
- 合并多个有序序列 → 最小堆多路归并

content_type: 题型模式
tag: 栈
title: 括号匹配/表达式求值问题的题型识别与算法选择

【题型特征】
题目涉及括号匹配、表达式求值、下一个更大元素、柱状图最大矩形。关键词包括：有效括号、计算器、下一个更大元素、接雨水、最大矩形。

【推荐算法】
1. 匹配栈：遇到左括号入栈，遇到右括号与栈顶匹配。适用于括号验证、HTML标签匹配。
2. 单调栈：维护栈内元素单调递增或递减，用于"下一个更大/更小元素"类问题，O(n)。
3. 表达式栈：用两个栈（操作数栈 + 运算符栈）实现中缀表达式求值，处理优先级和括号。

【判断依据】
- 括号/配对匹配 → 匹配栈
- 下一个更大/更小元素、接雨水、最大矩形 → 单调栈
- 表达式求值/计算器 → 双栈法
