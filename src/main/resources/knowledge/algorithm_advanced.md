tag: 动态规划,完全背包
title: 完全背包问题-物品可重复选取

【核心思想】
与 0-1 背包不同，完全背包中每种物品可以选取无限次。状态转移方程：dp[j] = max(dp[j], dp[j - w[i]] + v[i])，其中 j 从 w[i] 正序遍历（而非逆序），这样同一物品可以被多次选入。

【适用场景】
零钱兑换（硬币可重复使用）、完全背包求方案数、无限物品装箱问题。

【与 0-1 背包的区别】
0-1 背包内层循环逆序遍历容量，保证每个物品只选一次；完全背包正序遍历，允许同一物品多次选取。这是唯一的代码差异，但语义完全不同。

tag: 动态规划,区间DP
title: 区间DP-合并类问题的状态设计

【核心思想】
区间 DP 用 dp[i][j] 表示区间 [i, j] 的最优解。枚举分割点 k，将区间分为 [i, k] 和 [k+1, j] 两部分，合并子问题的解。遍历顺序：按区间长度从小到大枚举。

【经典问题】
矩阵链乘法、戳气球、石子合并、最长回文子序列。

【模板要点】
外层循环枚举区间长度 len 从 2 到 n，内层枚举左端点 i，右端点 j = i + len - 1，最内层枚举分割点 k 从 i 到 j-1。时间复杂度 O(n³)。

title: 状态压缩DP-用二进制表示集合状态

【核心思想】
当问题涉及"从 n 个元素中选子集"且 n 较小（n ≤ 20），可以用一个整数的二进制位表示哪些元素已被选取。dp[mask] 表示已选集合为 mask 时的最优解。

【适用场景】
旅行商问题（TSP）、任务分配问题、棋盘覆盖、集合划分。

【关键操作】
检查第 i 位：(mask >> i) & 1。设置第 i 位：mask | (1 << i)。清除第 i 位：mask & ~(1 << i)。枚举子集：for (int sub = mask; sub > 0; sub = (sub - 1) & mask)。

tag: 动态规划,树形DP
title: 树形DP-在树结构上自底向上递推

【核心思想】
在树上定义状态 dp[node][...]，通过 DFS 后序遍历，先计算子节点的 DP 值，再合并到父节点。常见状态设计：dp[node][0/1] 表示当前节点选或不选时的最优解。

【经典问题】
没有上司的舞会（树上最大独立集）、树的直径、树上背包、换根 DP。

【实现要点】
用邻接表存树，DFS 遍历时对每个子节点递归后更新当前节点的 DP 值。注意记录父节点避免回访。换根 DP 需要两次 DFS：第一次自底向上，第二次自顶向下。

title: 数位DP-统计满足条件的数字个数

【核心思想】
数位 DP 用于解决"在 [L, R] 范围内满足某种数位条件的数有多少个"类问题。将数字按位拆解，用 dp[pos][state][tight] 表示当前处理到第 pos 位、状态为 state、是否贴上界的方案数。

【适用场景】

content_type: 知识点
tag: 图论,最短路径,Dijkstra
title: Dijkstra算法-非负权图单源最短路径

【核心思想】
Dijkstra 算法用优先队列（最小堆）维护当前已知最短距离最小的未确定节点，每次取出堆顶节点并松弛其邻居。适用于所有边权非负的图。

【时间复杂度】
邻接表 + 优先队列实现：O((V + E) log V)。稠密图用邻接矩阵：O(V²)。

【注意事项】
不能处理负权边（负权边用 Bellman-Ford 或 SPFA）。Java 中用 PriorityQueue 实现时，注意已入队但距离已更新的节点需要跳过（懒删除）。

content_type: 知识点
tag: 图论,最短路径,Floyd
title: Floyd算法-全源最短路径

【核心思想】
Floyd 算法用三重循环计算任意两点间的最短路径：dp[i][j] = min(dp[i][j], dp[i][k] + dp[k][j])，其中 k 为中转点。外层循环枚举中转点 k，内层枚举起点 i 和终点 j。

【适用场景】
节点数较少（n ≤ 500）的全源最短路径问题、传递闭包、最小环检测。

【时间复杂度】
O(n³)，空间 O(n²)。可以处理负权边但不能处理负权环。

content_type: 知识点
tag: 图论,最小生成树
title: 最小生成树-Kruskal与Prim算法

【核心思想】
最小生成树（MST）是连通无向图中边权和最小的生成树。Kruskal 算法：将所有边按权值排序，依次加入不构成环的边（用并查集判环），适合稀疏图，O(E log E)。Prim 算法：从任意节点出发，每次选择连接已选节点和未选节点的最小权边，适合稠密图，O(V² 或 (V+E) log V)。

【适用场景】
网络布线最小成本、城市道路规划、聚类分析中的连通性问题。

tag: 图论,拓扑排序
title: 拓扑排序-有向无环图的线性排序

【核心思想】
拓扑排序将 DAG（有向无环图）的所有节点排成线性序列，使得对每条有向边 (u, v)，u 在序列中排在 v 之前。BFS 实现（Kahn 算法）：维护入度数组，将入度为 0 的节点入队，每次出队一个节点并将其邻居入度减 1，新的入度为 0 的节点入队。

【适用场景】
课程安排（先修课）、编译依赖、任务调度。如果最终排序结果的节点数少于图的总节点数，说明图中有环。

content_type: 知识点
tag: 图论,Bellman-Ford
title: Bellman-Ford与SPFA-含负权边的最短路径

【核心思想】
Bellman-Ford 对所有边进行 V-1 轮松弛操作，可以处理负权边并检测负权环。SPFA 是 Bellman-Ford 的队列优化版本，只对距离发生变化的节点的邻居进行松弛，平均时间复杂度 O(kE)（k 为常数），但最坏情况仍为 O(VE)。

【判断负权环】
Bellman-Ford 在第 V 轮松弛后如果仍有距离更新，则存在负权环。SPFA 中如果某节点入队次数超过 V 次，则存在负权环。

【核心思想】

【核心思想】
快速排序选择一个 pivot，将数组分为小于 pivot 和大于 pivot 两部分，递归排序。平均 O(n log n)，最坏 O(n²)（已排序数组 + 固定选首元素作 pivot）。快速选择（QuickSelect）只递归目标元素所在的一侧，平均 O(n) 找第 K 大/小。

content_type: 知识点
tag: 数学,组合数
title: 组合数计算-预处理与Lucas定理

【核心思想】
C(n, k) = n! / (k! × (n-k)!)。小范围（n ≤ 2000）用杨辉三角递推：C(n, k) = C(n-1, k-1) + C(n-1, k)。大范围取模用预处理阶乘和逆元：C(n, k) = fac[n] × inv[k] × inv[n-k] % MOD。超大范围（n > MOD）用 Lucas 定理。

【注意事项】
求逆元需要 MOD 为质数（费马小定理：a^(-1) = a^(MOD-2) % MOD）。预处理阶乘数组到 N 的时间和空间均为 O(N)。

title: 双指针进阶-多数之和与去重技巧

【核心思想】
三数之和：先排序，固定第一个数 nums[i]，对剩余部分用对撞双指针找两数之和等于 -nums[i]。四数之和类似，多加一层循环。时间复杂度 O(n²) 和 O(n³)。

content_type: 知识点
tag: 二分查找,二分答案
title: 二分答案-最小化最大值与最大化最小值

【核心思想】
当答案具有单调性时（答案越大越容易/越难满足条件），可以对答案范围进行二分搜索。每次取中间值 mid，用 O(n) 的 check 函数验证 mid 是否可行，根据结果缩小搜索范围。

【经典问题】
分割数组的最大值最小化、木材切割最大长度、运输货物最小载重。关键是识别出"最小化最大值"或"最大化最小值"的问题模式。

【模板要点】
确定答案的上下界 [lo, hi]，while (lo < hi) 循环中取 mid = lo + (hi - lo) / 2，check(mid) 为 true 则 hi = mid，否则 lo = mid + 1。
