title: TLE-超时常见原因与排查思路

【常见原因】
1. 暴力解法时间复杂度过高：O(n²) 或 O(n³) 在 n≥10^4 时极易超时，需考虑排序+双指针、哈希表、前缀和等优化
2. 递归/回溯未剪枝：搜索空间指数级增长，必须在递归入口处判断是否已不可能产生更优解
3. 死循环：二分查找边界更新不正确、while 条件永远为真、链表遍历未正确移动指针
4. 重复计算：递归中大量重叠子问题未记忆化，如斐波那契数列朴素递归 O(2^n)
5. IO 瓶颈：Java 中使用 Scanner 逐字符读取大量数据时远慢于 BufferedReader

【排查步骤】
- 先估算算法时间复杂度，对照数据范围判断是否可行（n≤10^5 一般需要 O(n log n) 以内）
- 检查循环和递归是否存在不收敛的情况
- 考虑是否有重复计算可以用记忆化或动态规划消除

tag: MLE
title: MLE-内存超限常见原因与排查思路

【常见原因】
1. 递归深度过大导致栈溢出：Java 默认栈大小约 512KB，递归深度超过数万层时会 StackOverflowError
2. 创建了过大的数组或集合：如 new int[10^8] 直接占用约 400MB，超出内存限制
3. 递归中每层创建新数组/字符串：如在回溯中每层 new ArrayList 复制当前路径，总内存为 O(n × 2^n)
4. 未及时释放引用：在循环中不断向 List 添加元素但从不清理，导致内存持续增长
5. 字符串拼接：循环中用 += 拼接字符串，每次创建新 String 对象，O(n²) 内存开销

【排查步骤】
- 估算数据结构占用内存：int 数组每元素 4 字节，对象数组每元素约 16~32 字节
- 递归深度 × 每层栈帧大小 是否超出栈限制
- 考虑用迭代替代递归，或用滚动数组压缩 DP 空间

tag: WA
title: WA-答案错误常见原因与排查思路

【常见原因】
1. 边界条件遗漏：空数组、单元素、全相同元素、最大/最小值等特殊输入未处理
2. 整数溢出：两个 int 相乘超过 2^31-1，需要用 long 存储中间结果
3. 浮点精度：浮点数比较不能用 ==，应使用 Math.abs(a-b) < 1e-9
4. 下标偏移：题目从 1 开始编号但代码从 0 开始，或反之，导致结果整体偏移
5. 排序不稳定：依赖排序后相同元素的相对顺序时，需使用稳定排序（如 Arrays.sort 对对象数组是稳定的，对基本类型不是）
6. 贪心策略错误：局部最优不等于全局最优，需要严格证明贪心正确性

【排查步骤】
- 用小规模样例手动模拟，对比预期输出
- 检查所有变量类型是否足够大（int vs long）
- 枚举边界 case：n=0, n=1, 全正/全负/含零

tag: RE
title: RE-运行时错误常见原因与排查思路

【常见原因】
1. 数组越界（ArrayIndexOutOfBoundsException）：循环条件写错、未检查数组长度、二维数组行列混淆
2. 空指针（NullPointerException）：链表遍历未判空、Map.get 返回 null 后直接拆箱、树节点为 null 时访问属性
3. 栈溢出（StackOverflowError）：递归无终止条件或终止条件不正确，导致无限递归
4. 除零错误（ArithmeticException）：取模或除法运算前未检查除数是否为零
5. 类型转换异常（ClassCastException）：泛型擦除后强转失败

【排查步骤】
- 仔细检查数组访问的下标范围，确保 0 ≤ index < length
- 所有可能为 null 的引用在使用前判空
- 递归函数必须有明确的 base case，且每次递归调用都在向 base case 靠近

tag: CE
title: CE-编译错误常见原因与排查思路

【常见原因】
1. 语法错误：缺少分号、括号不匹配、关键字拼写错误
2. 类型不匹配：将 String 赋给 int、泛型类型参数错误、方法返回类型与声明不一致
3. 变量未声明或作用域错误：在 if/for 块内声明的变量在块外使用
4. 导包缺失：使用 List、Map 等集合类未 import，使用 Arrays.sort 未 import java.util.Arrays
5. 主类名与文件名不一致：OJ 提交时类名必须为 Main（部分平台要求）

【排查步骤】
- 从编译器报错的第一行开始修复，后续错误可能是连锁反应
- 检查所有括号是否成对匹配
- 确认类名、方法签名、返回类型是否一致
