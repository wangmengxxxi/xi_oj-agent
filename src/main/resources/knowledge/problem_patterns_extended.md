tag: 双指针
title: 双指针问题的题型识别与算法选择

【题型特征】
题目涉及有序数组中的配对查找、链表操作、原地去重/移除。关键词包括：两数之和（有序）、三数之和、移除元素、合并有序数组、接雨水、盛最多水的容器。

【推荐算法】
1. 对撞指针：左右两端向中间逼近，适用于有序数组中的配对查找，O(n)
2. 快慢指针：两个指针同向不同速，适用于链表找环、找中点、原地去重
3. 滑动窗口指针：本质也是双指针，但侧重于连续子数组/子串问题

【判断依据】
- 有序数组找两数之和/配对 → 对撞指针
- 链表环检测/找中点 → 快慢指针
- 原地修改数组（去重/移除）→ 快慢指针

content_type: 题型模式
tag: 位运算
title: 位运算问题的题型识别与算法选择

【题型特征】
题目涉及异或、与或非、二进制表示、不用加减乘除实现运算。关键词包括：只出现一次的数字、汉明距离、2的幂、子集枚举、位计数。

【推荐算法】
1. 异或消除：a ^ a = 0，用于找唯一出现一次的元素
2. 位掩码枚举：用 n 位二进制数表示 n 个元素的选取状态，枚举所有子集
3. Brian Kernighan 算法：n & (n-1) 消除最低位的 1，用于统计二进制中 1 的个数

【判断依据】
- 找唯一/成对出现的元素 → 异或
- 枚举子集/状态压缩 → 位掩码
- 不用算术运算实现加减 → 位运算模拟

content_type: 题型模式
tag: 数学
title: 数学规律问题的题型识别与算法选择

【题型特征】
题目涉及数论、几何、概率、找规律。关键词包括：质数、最大公约数、幂运算、阶乘、组合数、取模、回文数、丑数。

【推荐算法】
1. 数论算法：GCD/LCM、质数筛、快速幂、模逆元
2. 找规律：手算前几项，观察递推关系或周期性
3. 数学公式：直接套用已知公式（如等差/等比数列求和、卡特兰数）

【判断依据】
- 涉及质数/因数 → 质数筛 + 分解
- 大数取模 → 快速幂
- 组合计数 → 预处理阶乘 + 逆元
- 看起来像 DP 但数据范围极大（n > 10^9）→ 找数学规律或矩阵快速幂

content_type: 题型模式
tag: 模拟
title: 模拟题的题型识别与解题策略

【题型特征】
题目描述一个具体的操作过程，要求按步骤执行并输出结果。没有明显的算法套路，重点在于准确理解题意和细心实现。关键词包括：螺旋矩阵、旋转图像、生命游戏、字符串处理、进制转换。

【解题策略】
1. 仔细读题，用小样例手动模拟一遍确认理解正确
2. 将操作步骤拆解为清晰的代码逻辑，避免一步到位导致混乱
3. 特别注意边界条件和特殊情况

【常见陷阱】
- 修改数组时影响后续读取（需要用临时数组或标记）
- 循环边界差一（off-by-one）
- 题意理解偏差（多读几遍题目描述）

content_type: 题型模式
tag: 字符串匹配
title: 字符串匹配问题的题型识别与算法选择

【题型特征】
题目要求在文本中查找模式串、判断子串关系、处理通配符或正则匹配。关键词包括：字符串匹配、模式搜索、通配符、正则表达式、最长公共前缀/后缀。

【推荐算法】
1. KMP 算法：构建 next 数组避免回溯，O(n+m) 精确匹配
2. 字符串哈希：O(1) 比较子串，适用于多模式匹配和重复子串检测
3. 动态规划：通配符匹配（* 和 ?）和正则表达式匹配用 DP 解决

【判断依据】
- 精确子串匹配 → KMP 或字符串哈希
- 通配符/正则匹配 → DP
- 多模式匹配 → AC 自动机或字符串哈希
- 最长回文子串 → 中心扩展或 Manacher

content_type: 题型模式
tag: 分治
title: 分治问题的题型识别与算法选择

【题型特征】
问题可以分解为若干规模更小的同类子问题，子问题的解可以合并为原问题的解。关键词包括：归并排序、快速排序、最近点对、大整数乘法、逆序对。

【推荐算法】
分治法：将问题分成两半，递归求解，合并结果。时间复杂度由 Master 定理分析。

【判断依据】
- 问题可以自然地一分为二 → 分治
- 合并阶段可以利用子问题的有序性 → 归并排序变体
- 与分治区分：DP 的子问题有重叠，分治的子问题互不相交

content_type: 题型模式
tag: 并查集
title: 连通性/等价关系问题的题型识别与算法选择

【题型特征】
题目涉及动态连通性判断、等价类合并、集合划分。关键词包括：朋友圈、连通分量、冗余连接、账户合并、等式方程。

【推荐算法】
并查集（Union-Find）：支持 O(α(n)) 近似 O(1) 的合并和查询操作。路径压缩 + 按秩合并是标准优化。

【判断依据】
- 动态添加边并判断连通性 → 并查集
- 静态图求连通分量 → DFS/BFS 也可以，但并查集更简洁
- 需要维护额外信息（距离、关系）→ 带权并查集

content_type: 题型模式
tag: 拓扑排序
title: 依赖关系/排序问题的题型识别与算法选择

【题型特征】
题目涉及任务依赖、课程先修关系、编译顺序等有向图的线性排序。关键词包括：课程表、任务调度、字典序最小的排列、判断是否有环。

【推荐算法】
拓扑排序（BFS/Kahn 算法）：维护入度数组，反复取出入度为 0 的节点。如果需要字典序最小，用优先队列替代普通队列。

【判断依据】
- 有向图判环 → 拓扑排序（排序结果节点数 < 总节点数则有环）
- 求合法的线性排序 → 拓扑排序
- 最长路径/关键路径 → 拓扑排序 + DP

content_type: 题型模式
tag: 树
title: 树形问题的题型识别与算法选择

【题型特征】
题目涉及二叉树或多叉树的遍历、路径、祖先、深度等。关键词包括：最近公共祖先、树的直径、路径总和、序列化与反序列化、层序遍历。

【推荐算法】
1. 递归（DFS）：大部分树问题的首选，自顶向下或自底向上递归
2. BFS（层序遍历）：按层处理节点，适用于求深度、层平均值、右视图
3. 树形 DP：需要在树上求最优解时，结合 DFS 后序遍历

【判断依据】
- 求路径/深度/祖先 → DFS 递归
- 按层处理 → BFS
- 树上最优解 → 树形 DP
- 二叉搜索树相关 → 利用 BST 中序有序性质

content_type: 题型模式
tag: 设计题
title: 数据结构设计题的题型识别与解题策略

【题型特征】
题目要求设计一个类，实现特定的数据结构或系统。关键词包括：实现 LRU 缓存、最小栈、用栈实现队列、设计哈希表、实现 Trie。

【解题策略】
1. 明确每个操作的时间复杂度要求
2. 选择合适的底层数据结构组合（如 HashMap + 双向链表实现 LRU）
3. 先写出接口骨架，再逐个实现方法

【常见组合】
- O(1) 插入删除查找 → HashMap
- O(1) 获取最值 → 辅助栈/堆
- O(1) 随机访问 + O(1) 删除 → HashMap + 数组（交换删除）
- 有序操作 → TreeMap/TreeSet
