title: 链表常用技巧-虚拟头节点与快慢指针

【核心思想】
虚拟头节点（dummy node）：在链表头部添加一个哨兵节点，统一处理头节点的插入和删除，避免大量 null 判断。快慢指针：slow 每次走一步，fast 每次走两步，用于找中间节点（fast 到末尾时 slow 在中间）、检测环（快慢指针相遇则有环）、找环入口（相遇后一个指针回到头，两个同速前进再次相遇即为入口）。

【OJ 高频操作】
反转链表（迭代三指针法）、合并两个有序链表、删除倒数第 N 个节点（快指针先走 N 步）、判断回文链表（找中点 + 反转后半段 + 比较）。

tag: 二叉搜索树,BST
title: 二叉搜索树-性质与中序遍历

【核心思想】
BST 的核心性质：左子树所有节点值 < 根节点值 < 右子树所有节点值。中序遍历 BST 得到的是严格递增序列。利用这个性质可以将很多 BST 问题转化为有序数组问题。

【OJ 高频操作】
验证 BST（中序遍历检查递增性，或递归传递上下界）、BST 中第 K 小元素（中序遍历计数）、BST 转有序双向链表、恢复 BST（中序遍历找两个逆序节点）。

【注意事项】
验证 BST 时不能只检查 node.val > node.left.val && node.val < node.right.val，必须确保左子树所有节点都小于根，右子树所有节点都大于根。

content_type: 知识点
tag: 堆,优先队列
title: 堆的实现原理与PriorityQueue使用

【核心思想】
堆是一种完全二叉树，分为最大堆（父 ≥ 子）和最小堆（父 ≤ 子）。用数组存储，节点 i 的左子节点为 2i+1，右子节点为 2i+2，父节点为 (i-1)/2。插入和删除操作通过上浮（sift up）和下沉（sift down）维护堆性质，时间 O(log n)。建堆 O(n)。

【Java PriorityQueue】
默认最小堆。最大堆用 new PriorityQueue<>(Comparator.reverseOrder()) 或 new PriorityQueue<>((a, b) -> b - a)。注意：PriorityQueue 的 remove(Object) 是 O(n) 不是 O(log n)。

tag: 树状数组,BIT
title: 树状数组-高效前缀和与单点修改

【核心思想】
树状数组（Binary Indexed Tree）支持 O(log n) 的单点修改和前缀和查询。核心操作 lowbit(x) = x & (-x) 取最低位的 1。update(i, val) 从 i 开始向上更新：i += lowbit(i)。query(i) 从 i 开始向下累加：i -= lowbit(i)。

【与线段树的对比】
树状数组代码量是线段树的 1/3，常数更小，但只能处理满足结合律和可减性的操作（如求和）。区间最值需要用线段树。

【OJ 应用】
动态前缀和、逆序对计数（离散化后从右往左插入，每次查询比当前值小的已插入元素个数）、区间修改 + 单点查询（差分树状数组）。

tag: 并查集,进阶
title: 并查集进阶-带权并查集与种类并查集

【核心思想】
带权并查集：在路径压缩时维护节点到根的权值（距离、关系等）。find 操作中累加路径上的权值，union 操作中根据两个节点到各自根的权值差计算新边的权值。种类并查集：将每个元素拆成 k 个虚拟节点表示 k 种身份，通过合并不同身份的虚拟节点表达"敌人的敌人是朋友"等关系。

【经典问题】
食物链（三种类型的环形关系）、银河英雄传说（带权并查集维护队列中的位置）、判断二分图（种类并查集或染色 BFS）。

tag: 单调队列
title: 单调队列-滑动窗口最值问题

【核心思想】
单调队列（双端队列）维护窗口内元素的单调性。以求滑动窗口最大值为例：队列存储下标，从队尾入队时弹出所有比当前值小的元素（维护递减），从队头弹出超出窗口范围的元素。队头始终是当前窗口的最大值。

【时间复杂度】
每个元素最多入队出队各一次，总时间 O(n)。

【OJ 应用】
滑动窗口最大值/最小值、多重背包的单调队列优化、DP 转移中的区间最值优化。

tag: 哈希表,设计
title: 哈希表设计与冲突处理

【核心思想】
哈希表通过哈希函数将 key 映射到数组下标，实现 O(1) 平均时间的插入、查找和删除。冲突处理：链地址法（每个桶挂链表，Java HashMap 采用此法，链表长度 > 8 转红黑树）、开放寻址法（线性探测、二次探测、双重哈希）。

【OJ 常用模式】
两数之和（边遍历边查表）、字母异位词分组（排序后的字符串作 key）、最长连续序列（HashSet 只从序列起点开始计数）、前缀和 + HashMap 求子数组和为 K。

content_type: 知识点
tag: 栈,表达式求值
title: 双栈法实现表达式求值

【核心思想】
用两个栈分别存储操作数和运算符。遍历表达式：遇到数字压入操作数栈；遇到运算符，先弹出栈中优先级 ≥ 当前运算符的运算符并计算，再将当前运算符入栈；遇到左括号入栈，遇到右括号弹出并计算直到左括号。

【优先级规则】
乘除 > 加减。括号内的运算符优先级最高。一元负号需要特殊处理（在表达式开头或左括号后出现的负号）。

【OJ 应用】
基本计算器系列（LeetCode 224/227/772）、逆波兰表达式求值、中缀转后缀表达式。

tag: 图论,二分图
title: 二分图判定与匹配

【核心思想】
二分图：图的所有节点可以分成两组，使得每条边的两个端点分属不同组。判定方法：BFS/DFS 染色法，交替染黑白两色，如果相邻节点同色则不是二分图。最大匹配：匈牙利算法，O(VE) 时间找最大匹配数。

【适用场景】
任务分配（人和任务的匹配）、棋盘覆盖（黑白格交替）、最小顶点覆盖（König 定理：二分图最小顶点覆盖 = 最大匹配）。

tag: 数据结构,LRU
title: LRU缓存-哈希表+双向链表实现

【核心思想】
LRU（Least Recently Used）缓存淘汰最久未使用的元素。用 HashMap 存储 key 到链表节点的映射，双向链表维护访问顺序（最近访问的在头部，最久未访问的在尾部）。get 和 put 操作都是 O(1)。

【实现要点】
双向链表需要 dummy head 和 dummy tail 简化边界处理。put 时如果容量已满，删除链表尾部节点并从 HashMap 中移除。get 时将访问的节点移到链表头部。Java 中可以用 LinkedHashMap 直接实现。
